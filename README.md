## How I Approach Problems
1. Brute Force First: Draft a working solution using intuitive logic (even if it uses higher time complexity like O(N^2)).
2. Refactor for Performance: Identify performance leaks (like unnecessary nested loops or repeated calculations) and optimize down to O(N) or O(1) where possible.
3. Clean Code: Format variables and logic structures so the code is highly readable and maintainable.
